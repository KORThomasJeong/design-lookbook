# design-lookbook — AI 작업 지침

AI가 웹 프로그램을 만들 때 참조하는 **번호 기반 디자인 컨셉 라이브러리**.
"3번(토스풍)으로 만들어줘", "2-b로 대시보드 짜줘" 같은 요청이 작동하는 것이 이 저장소의 존재 이유다.

## 핵심 원칙

1. **스펙이 원본, 갤러리는 뷰.** `presets/*/SPEC.md`와 `tokens.css`가 source of truth.
   `docs/`(갤러리)는 그 스펙의 레퍼런스 구현일 뿐이다. 둘이 어긋나면 스펙을 기준으로 갤러리를 고친다.
2. **번호 = 컨셉.** `1` Frost(Fluent풍), `2` Steel(Carbon풍), `3` Jelly(토스풍), `4` Obsidian(Linear/Vercel풍).
   색 변형은 `3-a`, `3-b`처럼 문자 접미사, 랜덤 팔레트는 `3-r<hue>`(재현 가능).
   전체 목록은 `presets/INDEX.md`가 유일한 레지스트리다.
3. **코드네임 + 영감 출처 명기.** 프리셋 공식 이름은 자체 코드네임(재질 시리즈: Frost, Steel, Jelly, Obsidian)을 쓰고,
   각 SPEC.md 머리에 `inspired by`를 한 줄 남긴다. 상표명을 파일/폴더명에 쓰지 않는다.
4. **풀 디자인 언어.** 프리셋 하나 = 토큰 + 컴포넌트 규칙 + 레이아웃 레시피 + 모션 + 보이스&톤 + 금지 규칙.
   토큰만 있는 프리셋은 미완성이다. AI 티가 나는 원인은 색이 아니라 간격 리듬·위계·모션·카피이기 때문.
5. **빌드 없음.** 갤러리와 데모는 순수 HTML/CSS/JS. 프레임워크·빌드 파이프라인 금지.
   토큰은 CSS custom properties가 원본이고, Tailwind v4/React 매핑은 스펙 안의 부록이다.
6. **모바일 퍼스트.** 모든 데모 화면은 390px 기준으로 먼저 설계하고 데스크톱으로 확장한다.

## 파일 지도

| 경로 | 역할 |
|---|---|
| `presets/INDEX.md` | 번호 레지스트리 + 각 프리셋 상태 (여기서 시작) |
| `presets/<번호>-<코드네임>/SPEC.md` | 프리셋의 풀 디자인 언어 (포맷: `SPEC-FORMAT.md`) |
| `presets/<번호>-<코드네임>/tokens.css` | CSS 변수 원본. 변형은 `[data-variant="b"]` 오버라이드 |
| `SPEC-FORMAT.md` | SPEC.md가 반드시 채워야 할 섹션 정의 (스키마) |
| `WORKFLOWS.md` | 요청 유형별 실행 절차 (추가/변형/랜덤 색/소비/데모) |
| `docs/` | GitHub Pages 갤러리. `docs/index.html`이 홈, `docs/<번호>-<코드네임>/`이 데모 |

## 요청이 들어오면

- **"N번으로/N-x로 만들어줘"** → `WORKFLOWS.md`의 W4(소비) 절차.
  INDEX.md에서 번호 해석 → 해당 SPEC.md + tokens.css 전체를 읽고 → 금지 규칙까지 지켜서 생성.
- **"OO 사이트 분석해서 프리셋 추가해줘"** → W1(신규 컨셉 추가) 절차. SPEC-FORMAT.md의 모든 섹션을 채워야 완료.
- **"N번에 색 변형/랜덤 색 추가"** → W2/W3. 각 SPEC.md의 §2.4(팔레트 생성 규칙) 범위 안에서만 색을 만든다.
- **"데모/갤러리 만들어줘"** → W5/W6. 컨셉당 5화면 풀 앱, 프리셋별 맞춤 시나리오(INDEX.md에 기재).

## 검증 (완료 선언 전)

각 SPEC.md의 **§12 채점 루브릭**이 판정 기준이다. 결과물(데모든 외부 적용이든)은 루브릭으로 자가검수하고,
사용자가 "될 때까지"를 원하면 loop-runner(자기개선 루프)에 §12를 합격 기준으로 넘겨 반복한다.
루브릭 공통 최소선:

- [ ] 본문 텍스트 대비 4.5:1 이상, 대형 텍스트/UI 3:1 이상 (스펙의 대비 규칙 준수)
- [ ] 간격이 해당 프리셋의 스페이싱 스케일 값만 사용 (임의 px 금지)
- [ ] SPEC.md §10 금지 규칙 위반 없음
- [ ] 데모는 모바일(390px)과 데스크톱(1280px) 양쪽에서 확인
- [ ] 새 프리셋/변형은 `presets/INDEX.md`에 등록됐는가

## 로컬 스킬

`/lookbook <번호[-변형]>` — **소비 전용**(W4)만 스킬화한다. 컨셉 추가·업그레이드(W1/W7)는
이 저장소 안에서 이 CLAUDE.md와 WORKFLOWS.md를 따라 수행하며 별도 스킬을 만들지 않는다.

## 현재 상태 / 진행 방식

**수직 슬라이스 전략**: 3번(Jelly, 토스풍)을 스펙→토큰→5화면 데모→갤러리까지 끝까지 완성해
"3번으로 만들어줘"가 실제 작동하는지 검증한 뒤, 그 포맷을 확정하고 1·2·4번을 병렬 제작한다.
포맷 변경이 필요하면 반드시 SPEC-FORMAT.md를 먼저 고치고 프리셋에 반영한다.
